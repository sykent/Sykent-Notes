## git 提交规则

* chore: 项目初始化 `git commit -m 'chore: 项目搭建'`
* feat: 增加新功能 `git commit -m 'feat: xx'`
* fix: 改bug `git commit -m 'fix: xx'`
* style： 格式（不影响代码运行的变动）
* test: 部署测试环境 `git commit -m 'test: xx'`
* refactor：重构（即不是新增功能，也不是修改bug的代码变动）
* docs：文档（documentation）
